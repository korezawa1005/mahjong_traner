# ■サービス概要
麻雀の「何切る」問題を通して、ユーザーの『雀力』を診断できるWebアプリです。
14枚の手牌から1枚を選んで回答し、その選択から複数の能力カテゴリをスコア化します。
診断結果はレーダーチャートで表示され、自分の長所・短所を可視化できます。

---
# ■このサービスへの思い・作りたい理由
私は麻雀が好きで、何切る問題を通じて自分の『雀力』を客観視できるものがあれば面白いのでは？と思うことがありました。
しかし、既存の何切る問題集は「正解・不正解」にとどまるものが多く、学びや成長につながる仕組みが少ないと感じていました。
そこで、自分自身や他のプレイヤーが「牌効率が得意だけど押し引きが弱い」など、自分のタイプを知ることができるアプリがあったら楽しいし、麻雀の上達にもつながると思い制作を決めました。

---
# ■ ユーザー層について
カジュアル麻雀プレイヤー    
→ オンライン麻雀（天鳳、雀魂、MJ）をプレイしていて「もっと上達したい」と思っている層。
雀荘勤務やプロを目指す真剣層    
→ 打ち筋の分析・研究に使えるよう、チャートによる診断結果が参考になる設計にしています。
SNSで麻雀ネタを発信している層
→ 問題や診断結果をSNSでシェアして、コミュニティ内で話題になることを想定。

---
# ■ サービスの利用イメージ

ユーザーはアプリにアクセスし、カテゴリ毎に何切るクイズを選択してから問題を解いて行きます。
画面に表示された14枚の手牌から、ユーザーが1枚を選んで切ると、回答の内容に応じてスコアが蓄積されます。
ユーザーは10問の「何切る」問題に回答すると、5つのカテゴリごとに自分の傾向がレーダーチャートで表示されます。これにより、自分の麻雀（打ち筋）の強みと弱みが視覚的にわかります。診断結果をもとに、足りていない部分を重点的に学習し、効率よく雀力を向上させることができます。

類似サービスとの違い

| 項目 | 他サービス（例：麻雀IQテストなど） | 本アプリの特徴 |
| --- | --- | --- |
| 問題構成 | 複数ジャンルが混ざった総合問題 | 牌効率・押し引き・リーチ判断・鳴き判断・手役意識など、カテゴリ別に出題可能 |
| 学習目的 | 一度きりのエンタメ寄り診断 | 特定のジャンルに集中してトレーニング可能。弱点克服を目的とした構成 |
| 結果の可視化 | 総合的なチャート表示のみ | ジャンルごとのスコアを個別にチャート表示し、自分の特徴が明確に分かる |
| 結果の蓄積 | 履歴機能がない or 限定的 | ログインユーザーごとに診断結果を保存し、成長を可視化できる |
| 拡張性 | 出題内容の更新が限られる | 問題投稿や成績の推移表示など、継続利用を促す機能を備えている |

本アプリの推しポイント
- 大人気麻雀アプリ 雀魂（1000万人）でラス回避率13位のH氏、麻雀全国チェーン店『ZOO』でランキング1位（会員約4000人）を取ったことのあるアプリ制作者の二人で監修 
- カテゴリ別に問題を選べるため、効率的に自分の弱点を鍛えられる
- 回答内容に応じてスコアを自動集計し、ジャンルごとに成績をレーダーチャートで可視化
- 診断結果を保存して後から見返すことができるため、成長を実感しながら学習を継続できる
- 出題から結果まで直感的に操作できる、実戦感のあるUI
- 初心者から中上級者まで、自分のペースで継続的にスキルアップできる設計

■ 機能候補

MVP（最小リリース時に実装したい機能）
- ユーザー登録・履歴保存
- 14枚の手牌から1枚を選ぶ問題出題機能
- 解答とカテゴリに応じたスコア蓄積
- 全10問終了後にレーダーチャートで診断結果を表示
- 問題ごとの簡単な解説表示
- 問題データ（50問：5カテゴリ×10問）のseed登録

本リリース・拡張機能（今後の予定）

- 問題投稿・編集機能（管理者 or 一般ユーザー）
- SNSシェア機能
- 正答率・選択率の集計（ランキング）
- 自分の総合的な成績の推移を見るページ（折れ線グラフなど）

---

機能の実装方針予定

| 機能 | 実装イメージ |
| --- | --- |
| **レーダーチャート表示** | `react-chartjs-2` を使い、診断5カテゴリを点数化してチャート描画 |
| **牌の表示と選択処理** | PNG/SVGの牌画像素材を表示し、クリックされた牌の index or tile_id をuseStateで管理 |
| **問題データの提供** | Railsの処理をRails APIを使用して、React側で使用できるようにする。処理としてはカテゴリごとにランダム出題する内容。問題はこちら側で作ります。 |
| **スコア集計処理** | フロントで選択ごとにカテゴリ別スコアを記録 → 結果ページで平均化して表示 |
| **解説付き回答結果** | 問題に`recommended_tile_id`と`commentary`を付与し、解答後に表示 |
| **CORS対応・API認証** | フロントからのアクセスに `rack-cors` を設定。将来的にJWTでセッション管理を導入予定 |
